# 开发检查清单

## 🔍 代码修改后必须执行的检查流程

### 1. **TypeScript 类型检查**
```bash
npm run typecheck
```
- ✅ 确保没有类型错误
- ✅ 检查新增的类型定义
- ✅ 验证导入/导出的类型匹配

### 2. **构建测试**
```bash
npm run build
```
- ✅ 确保项目能正常构建
- ✅ 检查是否有构建警告
- ✅ 验证所有路由和API端点

### 3. **代码质量检查**
```bash
npm run check
```
- ✅ 代码格式化检查
- ✅ ESLint 规则验证
- ✅ 代码一致性检查

### 4. **关联文件检查**

#### 当修改共享工具函数时：
- ✅ 检查所有导入该函数的文件
- ✅ 验证函数签名是否兼容
- ✅ 测试边界情况和错误处理

#### 当修改类型定义时：
- ✅ 检查使用该类型的所有组件
- ✅ 验证类型的向前/向后兼容性
- ✅ 更新相关的接口和实现

#### 当修改API端点时：
- ✅ 检查调用该API的前端代码
- ✅ 验证请求/响应格式
- ✅ 更新相关的错误处理

### 5. **数据库相关检查**

#### 当修改数据库schema时：
```bash
npm run db:generate
npm run db:push
```
- ✅ 生成新的Prisma客户端
- ✅ 验证数据库迁移
- ✅ 检查受影响的服务函数

#### 当修改数据服务时：
- ✅ 检查混合服务的兼容性
- ✅ 验证本地存储的回退逻辑
- ✅ 测试数据同步功能

### 6. **用户界面检查**

#### 当修改组件时：
- ✅ 检查组件的props类型
- ✅ 验证状态管理的一致性
- ✅ 测试组件的渲染逻辑

#### 当修改样式时：
- ✅ 检查响应式设计
- ✅ 验证主题一致性
- ✅ 测试不同屏幕尺寸

### 7. **错误处理检查**

#### 当修改错误处理逻辑时：
- ✅ 检查所有相关的try-catch块
- ✅ 验证错误边界的处理
- ✅ 测试用户友好的错误消息

### 8. **性能影响评估**

#### 当添加新功能时：
- ✅ 检查包大小影响
- ✅ 验证渲染性能
- ✅ 测试加载速度

### 9. **浏览器兼容性**

#### 当使用新的Web API时：
- ✅ 检查浏览器支持情况
- ✅ 添加必要的polyfill
- ✅ 提供降级方案

### 10. **安全性检查**

#### 当处理用户输入时：
- ✅ 验证输入验证逻辑
- ✅ 检查XSS防护
- ✅ 测试SQL注入防护

## 🚨 常见错误类型及预防

### TypeScript 类型错误
- **问题**: `Property 'x' does not exist on type 'never'`
- **原因**: 类型推断失败，通常是联合类型处理不当
- **解决**: 明确类型断言和类型守卫

### 导入/导出错误
- **问题**: 模块找不到或循环依赖
- **原因**: 文件路径错误或架构设计问题
- **解决**: 检查路径，重构循环依赖

### 运行时错误
- **问题**: 运行时访问undefined属性
- **原因**: 缺少空值检查
- **解决**: 添加防御性编程

### 异步处理错误
- **问题**: Promise未正确处理
- **原因**: 缺少await或错误处理
- **解决**: 正确使用async/await和try-catch
